# Git Sync

변경사항을 스테이징, 커밋, push까지 한 번에 처리한다.

## 파라미터

`$ARGUMENTS`: 선택적 커밋 메시지. 제공 시 해당 메시지 사용, 미제공 시 자동 생성.

## 워크플로우

```
[1] 변경사항 수집 및 분석
       ↓
[2] 커밋 메시지 결정 (commit.md 참조)
       ↓
[3] git add .
       ↓
[4] git commit -m "..."
       ↓
[5] git push origin <현재브랜치>
       ↓
[6] 완료 보고
```

## 단계별 상세

### 1. 변경사항 수집

```bash
git status
git diff --cached
git diff
git branch --show-current
```

### 2. 커밋 메시지 결정

[commit.md](commit.md)의 메시지 생성 규칙 적용:
- $ARGUMENTS 존재 시 → 해당 메시지 사용
- $ARGUMENTS 미존재 시 → 변경사항 기반 자동 생성
- 접두사 추론: [commit-prefix-rules.md](commit-prefix-rules.md) 참조

### 3. 스테이징 및 커밋

```bash
git add .
git commit -m "접두사: 메시지 제목

- 상세 변경 내용 1
- 상세 변경 내용 2"
```

### 4. Push

```bash
git push origin <현재브랜치>
```

## 출력 형식

```
═══════════════════════════════════════
🔄 Git Sync 시작
═══════════════════════════════════════

[1/4] 변경사항 분석 중...

[3/4] 커밋 중...
      ✅ 커밋 완료

[4/4] Push 중...

      ✅ Push 완료

───────────────────────────────────────
✅ Git Sync 완료
───────────────────────────────────────
```

## 예시

### 인자 제공 시

```
로그인 기능 구현
```

결과:
```
feat: 로그인 기능 구현

- 로그인 폼 컴포넌트 추가
- 인증 API 연동
```

### 인자 미제공 시

변경 파일 분석 후 자동 메시지 생성.

## 주의사항

- 변경사항이 없으면 중단
- 커밋 후 push 실패 시 에러 보고
- 민감한 파일 포함 시 경고
- force push는 수행하지 않음
