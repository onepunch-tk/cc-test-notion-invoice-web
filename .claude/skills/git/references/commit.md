# Git Commit

변경사항을 분석하고 커밋 메시지를 생성하여 커밋한다.

## 파라미터

`$ARGUMENTS`: 선택적 커밋 메시지. 제공 시 해당 메시지 사용, 미제공 시 자동 생성.

## 워크플로우

### 1. 변경사항 수집

```bash
git status
git diff --cached
git diff
git branch --show-current
```

### 2. 변경사항 분석

수집된 정보로 핵심 요약:
- 어떤 파일이 변경되었는가?
- 어떤 종류의 변경인가? (추가/수정/삭제)
- 변경의 목적은 무엇인가?

### 3. 커밋 메시지 결정

**$ARGUMENTS 존재 시**: 해당 메시지를 본문으로 사용

**$ARGUMENTS 미존재 시**: 변경사항 기반 메시지 자동 생성

### 4. 접두사 추론

[commit-prefix-rules.md](commit-prefix-rules.md) 참조

추론 우선순위:
1. 브랜치 이름 (feature/*, fix/*, hotfix/*, docs/* 등)
2. 변경된 파일 경로/확장자
3. diff 내용 분석
4. 기본값: `feat:`

### 5. 커밋 실행

```bash
git add .
git commit -m "접두사: 메시지 제목

- 상세 변경 내용 1
- 상세 변경 내용 2"
```

## 커밋 메시지 형식

```
접두사: 핵심 요약 (1줄, 50자 이내 권장)

- [실제 작업 내용 1]
- [실제 작업 내용 2]
- ...
```

## 예시

### $ARGUMENTS 제공 시

입력: `로그인 버그 수정`

출력:
```
fix: 로그인 버그 수정

- 세션 만료 처리 로직 수정
- 에러 메시지 개선
```

### $ARGUMENTS 미제공 시

변경: `src/components/Header.tsx` 새 파일 추가

출력:
```
feat: Header 컴포넌트 추가

- 반응형 네비게이션 구현
- 로고 및 메뉴 아이템 렌더링
```

## 주의사항

- 커밋 전 변경사항이 없으면 커밋하지 않음
- 민감한 파일(.env, credentials 등) 포함 시 경고
- 한국어로 커밋 메시지 작성
